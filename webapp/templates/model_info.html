Explainability Methodology

Per-prediction explanations are generated using SHAP TreeExplainer when available, which computes each feature's exact contribution to that specific prediction. If SHAP is not installed, the dashboard falls back to the model's own global feature_importances_, signed by whether the input value sits above or below the training set's median for that feature. The fallback is clearly labeled as an approximation, never presented as a true SHAP value.

Failure Mode Classification

The trained model is a binary classifier (failure / no failure) — it does not predict a specific failure category. The "Failure Type" shown after each prediction is a separate, rule-based heuristic built from the AI4I 2020 dataset's own published domain thresholds (tool wear, heat dissipation, power, and overstrain limits). It is a transparent explanatory aid, not a second model output.
